# Guia de contribuição  

## 1- Como contribuir
### Qualquer pessoa pode contribuir. Para isso, clone o nosso repositório, e aplique a sua ideia. Após isso, abra um Pull Request explicando qual foi sua ideia aplicada seguindo algumas diretrizes explicadas a seguir

## 2- Fork e clone do repositório
### Para fazer a clonagem do nosso projeto, faça um fork (clone no GitHub) do nosso repositório e baixe essa cópia na sua máquina para que você faça modificações e possa fazer o Pull Request em seguida

## 3- Convenções de Branch
### Para a sua atualização do nosso projeto, é interessante que ela *seja feita numa nova branch* com um nome curto e descritivo sobre o que ela traz de inovação, que é feita com base na branch *develop*

## 4- Diretrizes para mensagens de commit
### Seu commit deve ter um *título descritivo* sobre o que ele propõe de novo e, caso necessário, uma descrição detalhando melhor o que foi feito

## 5- Diretrizes do Pull Request
### Toda contribuição feita que você acha que merece ser adicionada ao código fonte, abra um Pull Request *seguindo o template* feito, e requisitando a junção da sua branch com a branch *develop*

## 6- Diretrizes de Issues
### Se você deseja contribuir com uma nova issue, *siga o template* para criar uma, deixando um título claro e uma descrição melhor da ideia, além de sugestões sobre como aplicar
